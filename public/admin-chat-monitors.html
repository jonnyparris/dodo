Chat Monitors

Per-(owner, Google Chat space) Durable Objects. Each one polls the space and forwards allowlisted HUMAN messages to a dedicated CodingAgent brain session — which decides what to do, calls tools, and uses the chat_reply MCP tool to post back. The monitor itself never calls an LLM. Reads use the owner's cf-portal MCP refresh-token config; sends use the ARIA chat middleware via the brain.

Owner email is the user whose cf-portal MCP refresh token reads the chat. Posting an existing (owner, space) pair edits in place — dedupe state and the alarm are preserved.

Context mode

When set to recent, non-allowlisted HUMAN messages are forwarded to the brain session as background annotations. The brain sees them in its history (for cross-referencing) but they don't trigger new turns. Default off means the brain only ever sees allowlisted senders' messages.
