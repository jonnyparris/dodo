How Dodo Works

Dodo is a self-hostable platform for dispatching long-running autonomous coding agents. Each session is an isolated workspace with its own files, git repo, and conversation history.

Send
Send starts a prompt and streams the response into the chat as it arrives. The prompt keeps running server-side, so it can survive reconnects and Durable Object eviction.
Fork
Creates a copy of the current session (files + messages) so you can branch experiments without losing the original.
Abort
Cancels the currently running prompt.
Workspace Files
The agent's virtual filesystem. It can read, write, search, and edit files autonomously during conversations. Click directories to expand.
Git
Full git support — init, add, commit, branch, push/pull to GitHub/GitLab with configured tokens.
Tasks
A Kanban board. Add tasks, then use the action buttons on each card: Dispatch creates a new session and starts the agent working on it. Done marks it complete. You can also drag cards between columns, select multiple cards for batch dispatch, or delete/cancel tasks. Ctrl/Cmd+click to multi-select.
Cron Jobs
Schedule delayed or recurring prompts that run automatically.
Memory
Persistent key-value store with search. Survives across sessions.
Integrations
Connect external MCP servers (GitHub, Cloudflare, Sentry, etc.) to extend the agent's capabilities.
Session Settings
Manage sharing, permissions, browser tools (requires admin approval), and per-session MCP overrides.
Config
Switch LLM models and gateways. The model dropdown is populated from the available models on the OpenCode gateway.
Allowlist
Hosts that sandboxed code execution is allowed to reach. Unlisted hosts are blocked.
