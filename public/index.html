How Dodo Works

Dodo is an autonomous coding agent. Each session is an isolated workspace with its own files, git repo, and conversation history.

Send / Async
Send waits for the full response (streaming). Async dispatches work in the background — you'll get a push notification when it completes.
Fork
Creates a copy of the current session (files + messages) so you can branch experiments without losing the original.
Abort
Cancels the currently running async prompt.
Workspace Files
The agent's virtual filesystem. It can read, write, search, and edit files autonomously during conversations. Click directories to expand.
Git
Full git support — init, add, commit, branch, push/pull to GitHub/GitLab with configured tokens.
Tasks
A Kanban backlog. Add tasks, advance their status. Click "Work on this" to dispatch a task to a new session.
Prompts
History of async prompts with their status (queued, running, completed, failed, aborted).
Cron Jobs
Schedule delayed or recurring prompts that run automatically.
Memory
Persistent key-value store with search. Survives across sessions.
Integrations
Connect external MCP servers (GitHub, Cloudflare, Sentry, etc.) to extend the agent's capabilities.
Session Settings
Manage sharing, permissions, browser tools, approvals, and per-session MCP overrides.
Config
Switch LLM models and gateways. The model dropdown is populated from the available models on the OpenCode gateway.
Allowlist
Hosts that sandboxed code execution is allowed to reach. Unlisted hosts are blocked.
