How Dodo Works

Dodo is a self-hostable autonomous coding agent running on Cloudflare Workers and Durable Objects, built on @cloudflare/think.

Architecture

Life of a Request

Async Prompt Durability

Async prompts (POST /session/:id/prompt) run inside durable fibers. If the DO is evicted mid-prompt, the fiber replays from scratch on recovery. A stashFiber() checkpoint after the LLM call completes ensures that recovered prompts skip the expensive chat and go straight to finalization.

Session Branching

Forking creates a new Dodo session/DO with copied files and transcript. Think's internal SessionManager.fork() is not used for user-visible branching — Dodo snapshots handle this at the DO level. Snapshot v2 stores UIMessages with sidecar metadata; v1 import is supported for backward compatibility.

Features

FeatureDescription
SessionsIsolated workspaces with their own files, git, and message history. Create, list, delete, fork.
Streaming ChatSynchronous messages stream tokens via SSE. Async prompts run in fiber-backed background tasks with push notifications.
Think Chat LoopThe LLM autonomously reads/writes workspace files via @cloudflare/think's agentic loop with createExecuteTool.
WorkspaceVirtual filesystem backed by SQLite (via @cloudflare/shell). Full CRUD, search, and in-file replace.
Code ExecutionSandboxed JavaScript execution via Worker Loaders. Outbound network gated by the host allowlist.
GitFull isomorphic-git: init, clone, add, commit, branch, checkout, push, pull, diff. Auto-injects GitHub/GitLab tokens.
MCP Server/mcp endpoint exposes ~22 tools. Connect from OpenCode or any MCP client to dispatch work remotely.
TasksKanban backlog (backlog/todo/in_progress/done). Click to advance. Moving to "in progress" auto-dispatches a prompt.
CronSchedule delayed, cron, or interval tasks that fire prompts via the Agents SDK scheduler.
MemoryPersistent key-value store with text search and tags. Survives across sessions.
Model SelectorPer-session config via Think.configure(). Models populated from the OpenCode provider, cached for 24 hours.
Push NotificationsVia ntfy.sh on prompt completion, failure, or abort.
Token TrackingPer-message and per-session token usage via message_metadata sidecar.

Connecting via MCP

Add to your opencode.json:

{
  "mcp": {
    "dodo": {
      "type": "remote",
      "url": "https://dodo.jonnyparris.workers.dev/mcp",
      "headers": {
        "Authorization": "Bearer {file:.opencode/dodo-mcp-token}"
      }
    }
  }
}

Generate a token: openssl rand -base64url 32 > ~/.opencode/dodo-mcp-token

Set it as a secret: wrangler versions secret put DODO_MCP_TOKEN
