Create Your First Session

Create a session and send your first prompt

Click the + floating action button in the lower-right corner to create a new session. Dodo opens a fresh workspace and chat view.

Type a prompt into the chat box, then press Enter. Start with something concrete, like reviewing a file, adding a route, or explaining a test failure.

Watch the streamed response as the agent thinks, reads files, and edits code. You can follow the output live in the chat timeline.

Summarize this repo and tell me where the API routes live.

Tip: Give the agent one clear task per prompt. You will get better results than asking for several unrelated changes at once.

Clone a GitHub Repo

Bring a repository into the session workspace

Before cloning private GitHub repos, open Config > Secrets and add your token as GITHUB_TOKEN. Public repos can usually clone without it, but private repos require it.

You can ask the agent in chat to clone a repo for you, or it can use the git_clone tool directly inside the session workspace.

Clone https://github.com/user/repo

git_clone({ url: "https://github.com/user/repo", dir: "repo" })

Once the clone finishes, ask the agent to inspect files, run tests, or start making changes inside that workspace.

Warning: Set the token first if you plan to clone or push private repositories. Missing credentials usually show up as git auth errors.

Generate an Image

Create pictures without leaving the chat

Type / in the chat input to open the slash-command menu, pick /generate, then describe the image you want. Press Enter. Dodo sends the prompt to Workers AI FLUX-1-schnell and renders the image inline in a few seconds.

/generate a cyberpunk cat wearing sunglasses, neon city background

No LLM is involved — the slash command routes straight to the image model. That means the image is cheap, fast, and doesn't consume your chat model's context window.

From an MCP client, call generate_image directly with a session ID and a prompt. Sending a regular chat message whose content starts with /generate works too; the server detects and reroutes it.

generate_image({ sessionId: "session_123", prompt: "a steampunk dodo bird" })

Limits: prompts up to 2048 characters, 30 images per hour and 100 per day per user. Requires the AI binding in wrangler.jsonc — paid Workers plans include it.

Fork a Session for Experiments

Try risky changes without touching the original session

Open the session you want to preserve, then click the Fork button. Dodo creates a copy with the same workspace and conversation context.

Use the fork when you want to test a large refactor, a dependency upgrade, or a prompt that may take the work in a different direction.

If you are using MCP tools, call fork_session instead of clicking the button. The result is the same: a new session you can modify independently.

fork_session({ sessionId: "session_123" })

Tip: Name the fork clearly so you can tell the safe baseline from the experiment.

Schedule a Recurring Task

Run prompts on a cron schedule

Open the Cron tab to create a scheduled task. Choose a scheduling type — delayed (one-shot after N seconds), scheduled (one-shot at a specific datetime), cron (recurring cron expression), or interval (recurring every N seconds) — then enter the prompt you want Dodo to run.

Use this for repeated work like test runs, dependency checks, or status reports. Keep the prompt focused and action-oriented.

Type: cron
Cron: 0 * * * *
Prompt: Run the test suite, summarize failures, and open a task if anything regressed.

After saving, Dodo will dispatch the prompt on schedule. Review the resulting sessions to inspect logs, diffs, and outputs.

Tip: Start with hourly or daily schedules until you trust the prompt. Frequent jobs can create a lot of sessions quickly.

Connect an MCP Server

Add external tool servers through Integrations

Open the Integrations panel to connect external MCP servers such as GitHub, Sentry, or Cloudflare. These servers extend what the agent can do.

Enter the MCP server URL, then add an auth token if the server requires one. Save the integration so it becomes available to sessions.

URL: https://example.com/mcp
Token: optional-secret-token

Once connected, ask the agent to use those tools in chat. For example, you can request GitHub issue lookups, Sentry investigation, or Cloudflare changes.

Warning: Only connect MCP servers you trust. They can expose powerful external actions to the agent.

Dispatch Tasks from the Kanban Board

Turn tasks into autonomous agent sessions

Open the task board and create cards for the work you want to delegate. Keep each task small enough to describe in one focused prompt.

Select one or more tasks, then dispatch them to create autonomous agent sessions. Use Ctrl+click or Cmd+click to multi-select cards before batch dispatch.

Dodo starts one session per dispatched task, so you can work through several items in parallel and review each result independently.

Example task: Update the API docs for the new /howto route.

Tip: Batch dispatch works best when the selected tasks do not depend on each other.

Push Code to GitHub

Send workspace changes to a remote repository

First, add a git token in Config > Secrets if the remote requires authentication. Without credentials, push commands will fail.

Then ask the agent to commit and push your branch, or let it use git_push from the session workspace. Be explicit about the branch name when you care about the destination.

Commit my changes and push this branch to origin.

git_push({ remote: "origin", ref: "feat/my-branch" })

After the push completes, verify the branch on GitHub and open a pull request if you want review before merging.

Tip: Ask the agent to run tests before pushing so you do not publish a broken branch.

Use Memory Across Sessions

Store information that survives session deletion

Dodo includes a persistent memory store for long-lived notes, decisions, and small pieces of structured data. Memory survives even if you delete the session that wrote it.

Write key-value pairs you want to reuse later, such as deployment notes, repository conventions, or customer-specific details. Search works across all saved entries.

memory_write({ title: "dodo default branch", content: "main" })
memory_search({ query: "default branch for dodo" })

Use memory for durable context, not large source files. Keep entries short, searchable, and specific enough to help future sessions.

Tip: Save recurring setup facts in memory so you do not need to repeat them in every new session.

Add a Skill

Teach the agent a workflow once and reuse it everywhere

Skills are markdown files that describe a workflow, format, or domain the agent should know about. Dodo's skill format is identical to Claude Skills and OpenCode skills, so any skill written for either tool runs in Dodo unchanged.

Three ways to add one:

  1. Personal skill — call skill_write from your MCP client. The skill is stored in your user account and appears in every new session.
  2. Workspace skill — drop a SKILL.md into .dodo/skills/<name>/, .claude/skills/<name>/, or .opencode/skill/<name>/ in the workspace (or in any cloned repo). It auto-loads on the next prompt.
  3. Import from URL — call skill_import_url({ url: "https://raw.githubusercontent.com/.../SKILL.md" }) to pull a skill from someone else's repo into your personal library.
skill_write({
  name: "release-checklist",
  description: "Pre-release checks for the dodo repo. Use before tagging a release.",
  body: "# Release checklist\n\n1. Run `npm test`\n2. Update CHANGELOG\n3. Tag and push"
})

Once added, the agent sees the name and description in every session. To use it, the agent calls the skill tool with the name and the full body loads on demand. Bundled files (references/, scripts/) are listed by path but only loaded if the agent reads them.

Tip: Write the description like a tool description — say what the skill does and when to use it. The agent only sees the description until it loads the body.
