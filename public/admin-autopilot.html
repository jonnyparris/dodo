Safety rails

  • Hard cap: 50 tool turns. Worker writes a status note and stops if stuck.
  • Draft PRs only. Human review always required.
  • Never force-pushes or rewrites history.
  • Skips changes that would touch more than 5 files.
  • Does not edit secrets or .dev.vars.

What's needed

The worker needs Cloudflare Logs access (CLOUDFLARE_API_TOKEN + CLOUDFLARE_ACCOUNT_ID + DODO_WORKER_NAME) and a GitHub credential reachable via the normal session integration flow. If logs aren't configured, the worker will return an empty result from fetch_worker_logs and likely stop without a PR.
