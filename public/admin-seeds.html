Seed Cache

Admin-owned warm clones of known repos. User runs fork these instead of cloning fresh, saving wall time and tokens. Seeds never auto-expire — clean them up here when you want a fresh start. Seeds are hidden from the regular session list.

About

The first call to get_or_create_seed_session for a given repo + base branch clones the repo into an admin-owned session and registers it here. Every subsequent call across all users forks that session (sub-second) instead of cloning. Refresh runs git pull on the seed so future forks pick up upstream changes. Delete removes the registry entry and the underlying session — the next caller will cold-clone a new seed.
